# ABLkit 学习笔记

> Fork 时间：2026-05

## 这是什么

ABLkit 是**溯因学习（Abductive Learning）** 的 Python 工具包。溯因学习是神经符号 AI 的一个具体范式——让机器学习（神经网络）和逻辑推理（符号系统）在同一个框架内协同工作。

## 核心思想

传统做法是「先训练模型，再规则校验」——两个阶段是割裂的。ABL 的思路是：

```
神经网络感知 → 产生伪标签 → 逻辑知识库验证 → 
    ↓ 不一致？
溯因推理修正 → 反馈给神经网络重训 → 循环直到收敛
```

类比：神经网络是「眼睛」（看图识别），逻辑知识库是「大脑」（判断对错），ABL 是两者之间的对话机制。

## 与 NSEAP 的关系

东方国信智能体系统部的 NSEAP（神经符号进化智能体平台）的核心挑战就是「神经+符号怎么协同」。ABLkit 提供了其中一种答案：

- **神经侧**：分类/感知模型（PyTorch/TensorFlow）
- **符号侧**：Prolog 逻辑知识库
- **桥接**：ABL 循环（伪标签 → 溯因 → 修正）

NSEAP 的「进化」部分可能需要类似机制——Agent 执行任务 → 规则引擎判定结果 → 不一致时推理原因 → 反馈给 Agent 调整策略。

## 关键代码路径

- `ablkit/learning/` — 学习模块（神经侧）
- `ablkit/reasoning/` — 推理模块（符号侧，基于 PySwip/Prolog）
- `ablkit/abduction/` — 溯因推理核心
- `examples/` — 手写等式识别等示例

## 学到的关键概念

1. **溯因（Abduction）≠ 演绎（Deduction）≠ 归纳（Induction）**
   - 演绎：规则 + 前提 → 结论（如果下雨，地会湿。下雨了。地湿了。）
   - 归纳：观察 → 规则（每次下雨地都湿了。所以下雨导致地湿。）
   - 溯因：观察 + 规则 → 最可能的前提（地湿了。下雨会让地湿。可能下雨了。）

2. **伪标签（Pseudo-label）**：神经网络的初始输出不准确，但可以作为「假设」交给逻辑系统验证，然后溯因修正

3. **不一致驱动的学习**：不是等到模型训练完再检查，而是在训练过程中就让逻辑系统介入修正

## 局限性

- 依赖 Prolog/PySwip，工程化部署有门槛
- 主要针对分类+规则验证场景，不是通用 Agent 框架
- 中文社区资料少，论文需要一定数学基础
